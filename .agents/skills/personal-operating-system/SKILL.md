# Personal Operating System

## Purpose

Maintain a lightweight operating system across goals, projects, areas, tasks, and reviews.

## Workflow

1. Separate goals, projects, areas, tasks, and references.
2. Identify active commitments and stale commitments.
3. Clarify priorities, deadlines, and constraints.
4. Create review cadence.
5. Suggest next actions.
6. Keep the system small enough to actually use.

## Output

- Weekly or monthly review.
- Project inventory.
- Priority plan.
- Next actions.
- Areas dashboard.
- Reflection prompts.

## Rules

- Prefer fewer active priorities.
- Distinguish commitments from ideas.
- Keep recommendations operational.
