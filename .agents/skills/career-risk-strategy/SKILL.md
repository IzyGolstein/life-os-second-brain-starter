# Career Risk Strategy

## Overview

Give concrete strategy under geopolitical and tool-access constraints. Avoid generic option-value advice; produce a compact answer plus an appendix with target lists when needed.

## Required Output

1. Reconstructed question and assumptions.
2. Direct recommendation.
3. Risk model: tool access, sanctions/compliance, visa, market, skill compounding.
4. Concrete options by relevant route: PhD, RA/pre-doc, industry, startup, research role, master's, bridge country, or other routes the question implies. Treat these as vehicles toward relocation, tool access, legal stability, signal, runway, and environment quality, not as isolated identities.
5. Candidate countries, programs, companies/labs, roles, salary/financing notes when relevant.
6. What to browse/read next.
7. Decision criteria and next step.

## Source Mix

Use official sources for eligibility, sanctions, visas, and program facts. Also use news/community sources when the question asks about lived access, blocks, VPN bans, Russia-based account risk, or local industry trends.

Examples:

- official provider docs for supported countries;
- Habr, vc.ru, Telegram/community reports, GitHub issues, forums for lived access signals;
- university/program pages for admissions/funding;
- salary sources and job postings for compensation;
- official sanctions pages for legal/compliance constraints.

## Quality Gates

- Did we check current sources when facts can change?
- Did we avoid only official docs when the question asks about actual blocking behavior?
- Did we give concrete programs/companies/countries?
- Did we separate final destinations from bridge locations?
- Did we mention what depends on the user's preferences and ask only necessary questions?
